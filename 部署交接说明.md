# SoundDrift 部署交接说明（给 Claude Code 看）

这是一个**音乐卡片展示网页**项目。封面/音频/歌词/MV 全部**实时从网易云音乐在线获取**（经本地 Node 代理转同源），无需打包大媒体文件。目标：在这台（公司）电脑本地把它跑起来，浏览器打开 `http://127.0.0.1:8080/` 使用，含网易云搜索功能。

---

## 一、架构（必须同时运行两个本地 Node 服务）

1. **网易云接口服务**（第三方包 `NeteaseCloudMusicApi`）
   - 监听 **端口 3000**
   - 入口：`netease/node_modules/NeteaseCloudMusicApi/app.js`
   - 启动：在 `netease/` 目录下 `set PORT=3000 && node node_modules/NeteaseCloudMusicApi/app.js`

2. **主站 + 代理服务**（`server.js`，纯 Node 原生 http，无第三方依赖）
   - 监听 **端口 8080**，`HOST=127.0.0.1`
   - 同时做：① 静态托管 `index.html` 等 ② `/ncm/*` 代理（把网易云的音频/MV/图片/接口转成同源，解决浏览器 Web Audio 跨域 + canvas 取色污染）
   - 代理内部请求 `NCM_API = http://127.0.0.1:3000`（即上面那个服务）
   - 启动时读取同目录 `ncm_cookie.txt`（网易云黑胶 VIP 登录态，用于解锁会员歌曲���曲；缺失则部分歌只有 30s 试听，不影响整体运行）

浏览器只访问 8080；8080 内部再调 3000。两个都得开，只开一个会导致音频/封面/歌词/MV 全部加载失败。

---

## 二、环境要求

- **Node.js**：装 LTS 版即可（本项目在 v24 下开发，v18+ 都行）。官网 https://nodejs.org
- **操作系统**：Windows（提供的 `.bat` 启动脚本是 Windows 批处理）
- **依赖**：`netease/node_modules/` 已随包带来，**通常不需要重装**。
  - 仅当启动报模块错误时，在 `netease/` 目录执行 `npm install`（需要网���能访问 npm registry；国内可加 `--registry=https://registry.npmmirror.com`）。
  - `server.js` 只用 Node 内置模块，无需 install。

---

## 三、启动方式

**最简单**：双击项目根目录的 `启动SoundDrift.bat`
- 它会：检测 Node → 起 3000 服务 → 起 8080 服务 → 等 4 秒 → 自动打开浏览器到 `http://127.0.0.1:8080/`
- 任务栏会有两个最小化的命令行窗口（服务进程，别关）

**手动启动**（脚本失效时）：开两个终端
```bat
:: 终端1
cd <项目目录>\netease
set PORT=3000
node node_modules\NeteaseCloudMusicApi\app.js

:: 终端2
cd <项目目录>
node server.js
```
然后浏览器开 `http://127.0.0.1:8080/`

**关闭**：双击 `关闭SoundDrift.bat`（按端口 kill 掉 8080/3000 的 node 进程）

## 四、验证（确认跑通）

依次确认（命令行用 curl 或浏览器都行）：
1. `http://127.0.0.1:8080/` 能打开页面（卡片圆弧、顶部视频在动）→ 8080 服务 OK
2. `http://127.0.0.1:3000/search?keywords=test&limit=1` 返回 JSON → 3000 服务 OK
3. `http://127.0.0.1:8080/ncm/audio?id=3382153693`（带 Range 请求）返回 206 + audio/mpeg → 音频代理 OK
4. 页面里点底部那条白竖线 → 出搜索框 → 输歌名回车 → 进度 0%→100% → 歌曲飞到中间播放、有封面/歌词/MV → 全链路 OK
5. 点中间卡片返回首页，卡片并入圆弧继续转

---

## 五、最关键的坑：公司网络必须能访问网易云

整个项目**强依赖网易云在线资源**。公司网络/防火墙若屏蔽了以下域名，会导致搜索/音频/封面/MV/歌词失败（页面能开但点开卡片没声音没图）：
- `music.163.com`、`interface.music.163.com`（接口）
- `*.music.126.net`（封面图）
- `*.vod.126.net`（音频/MV 流）

**第一步务必先测**：启动后访问 `http://127.0.0.1:3000/search?keywords=周杰伦&limit=1`，能返回带歌曲的 JSON 就说明公司网络通网易云；返回空/超时/报错则是被网络限制了，此时项目无法正常用（需换网络环境，本工具无法绕过公司防火墙）。

## 六、其他说明

- **端口占用**：若 8080 或 3000 被别的程序占用，启动会失败。可改 `server.js` 顶部 `PORT`（8080）和 `NCM_API`（3000）、以及启动脚本里的 `PORT=3000`，三处保持对应。
- **cookie 过期**：`ncm_cookie.txt` 是网易云登录态，数周后过期 → 会员歌曲(少数几首)恢复 30s 试听，其余正常。项目根有 `qrlogin.js`(生成扫码二维码 `ncm_login_qr.png`) 和 `qrcheck.js`(扫码后抓 cookie 写回 `ncm_cookie.txt`)，需要时在 3000 服务运行时 `node qrlogin.js` → 手机网易云扫 `ncm_login_qr.png` → `node qrcheck.js`。
- **不需要**：本项目不需要数据库、不需要构建打包、不需要 npm install（除非 netease 依赖缺失）、不需要公网/域名/服务器。纯本地双 Node 进程 + 浏览器。
- **字体**：`assets/fonts/` 里的字体是本地 @font-face 引用的（Fraunces 英文 / Cervanttis 专辑名 / AiDianFengYaHei 中文歌词），必须随包带上。
