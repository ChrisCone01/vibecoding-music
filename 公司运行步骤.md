# 公司电脑运行 SoundDrift —— 给 Claude Code 的任务清单

我（用户 Chris）从家里电脑把这个项目推到了 GitHub。现在我在**公司的 Windows 电脑**上，想把它跑起来用，浏览器打开 `http://127.0.0.1:8080/`。请帮我按下面步骤完成。

仓库：`https://github.com/ChrisCone01/vibecoding-music`

---

## 这是什么项目

音乐卡片展示网页。封面/音频/歌词/MV **全部实时从网易云音乐在线获取**（经本地 Node 代理转同源）。需要**同时运行两个本地 Node 服务**：
- **端口 3000**：网易云接口服务（第三方包 NeteaseCloudMusicApi，依赖已随仓库一起克隆下来，无需重装）
- **端口 8080**：主站 + `/ncm/*` 代理（`server.js`，纯 Node 内置模块，无依赖）

浏览器只访问 8080；8080 内部再调 3000。两个都得开。

---

## 请帮我做这些（按顺序）

### 第 0 步：确认 Node.js 已装
```
node -v
```
- 没输出版本号 → 帮我装 Node.js LTS（https://nodejs.org），装完继续。

### 第 1 步：克隆仓库（如果还没克隆）
```
git clone https://github.com/ChrisCone01/vibecoding-music.git
cd vibecoding-music
```
- 仓库已包含 `netease/node_modules`（依赖随仓库带来），**通常无需 npm install**。
- 万一 `netease/node_modules` 缺失或启动报模块错误，再执行：
  ```
  cd netease
  npm install --registry=https://registry.npmmirror.com
  cd ..
  ```

### 第 2 步：先验证公司网络能不能访问网易云（最关键）
先单独把网易云接口服务跑起来测一下：
```
cd netease
set PORT=3000
node node_modules\NeteaseCloudMusicApi\app.js
```
另开一个终端测：
```
curl "http://127.0.0.1:3000/search?keywords=%E5%91%A8%E6%9D%B0%E4%BC%A6&limit=1"
```
- **返回带歌曲的 JSON** → 公司网络通网易云，继续。
- **超时 / 空 / 报错** → 公司防火墙封了网易云（music.163.com、*.music.126.net、*.vod.126.net）。**此时项目无法使用，任何工具都绕不过公司防火墙**，请告诉我这个结果，需要换网络环境。

### 第 3 步：双击启动
- 双击项目根目录的 **`启动SoundDrift.bat`**（会自动起两个服务 + 开浏览器到 http://127.0.0.1:8080/）
- 或手动开两个终端：
  ```
  :: 终端1
  cd vibecoding-music\netease
  set PORT=3000
  node node_modules\NeteaseCloudMusicApi\app.js

  :: 终端2
  cd vibecoding-music
  node server.js
  ```
- 关闭用 `关闭SoundDrift.bat`

### 第 4 步：验证跑通
1. `http://127.0.0.1:8080/` 出现卡片圆弧、顶部视频在动
2. 点底部白竖线 → 搜索框 → 输歌名回车 → 进度 0%→100% → 歌曲飞到中间播放（有封面/歌词/MV/声音）
3. 点中间卡片返回首页

## 关于会员歌曲（可选，不做也能用）

有 4 首歌（LET ME LUV U、wintry rain/冷色晚风、Scared 2 Be Lonely、Manta）是网易云会员歌曲。**没有登录态时只播 30 秒试听并循环**；其余 16 首及搜索的大部分歌都能整首播放。

如果要让那 4 首也整首播放，需要在这台电脑**用我的网易云账号扫码登录一次**（登录态 `ncm_cookie.txt` 出于安全没有上传 GitHub）：
1. 确保 3000 服务在运行
2. 在项目根目录执行 `node qrlogin.js` → 生成二维码图片 `ncm_login_qr.png`
3. 打开 `ncm_login_qr.png`，用**手机网易云音乐 App** 扫码登录（需要我本人的手机/账号）
4. 执行 `node qrcheck.js` → 自动把登录态写入 `ncm_cookie.txt`
5. 重启 `server.js`（关掉重新 `node server.js`），那 4 首即可整首播放

> 不想登录就跳过，不影响整体使用。

## 常见问题

- **端口被占用**：8080 或 3000 被别的程序占了会启动失败。可改 `server.js` 顶部的 `PORT`(8080) 和 `NCM_API`(3000)、以及启动脚本里的 `PORT=3000`，三处对应。
- **页面能开但点开卡片没声音没图**：基本是公司网络访问不了网易云（见第 2 步）。
- **不需要**：数据库、构建打包、公网/域名/服务器都不需要。纯本地双 Node 进程 + 浏览器。
- 更详细的架构说明见同目录 `部署交接说明.md`。
